# 1. 헤르메스 에이전트 소개

> 공식 문서 인용 위주로 정리. 평가나 해석은 라이브에서 직접 따져봅니다.

## 한 줄 정의 (공식)

**헤르메스 에이전트(Hermes Agent)** — Nous Research가 공개한 오픈소스 에이전트. 공식 슬로건은 다음 한 문장입니다.

> **"The Agent That Grows With You."**
>
> — [hermes-agent.nousresearch.com](https://hermes-agent.nousresearch.com/)

자기 정의 한 문단(공식 홈페이지, 직접 인용):

> "Not a coding copilot tethered to an IDE or a chatbot wrapper around a single API. An autonomous agent that lives on your server, remembers what it learns, and gets more capable the longer it runs."

번역하면 — IDE에 묶인 코딩 copilot도 아니고, 단일 API를 감싼 챗봇 wrapper도 아니다. 본인 서버에 사는 자율 에이전트로서, 배운 것을 기억하고 오래 돌수록 더 유능해진다.

## 공식 6가지 특징 (homepage 그대로)

| 영문 (원문) | 의미 |
|---|---|
| **Lives Where You Do** | 본인이 쓰는 채널에 산다 — Telegram · Discord · Slack · WhatsApp · Signal · Email · CLI |
| **Grows the Longer It Runs** | 오래 돌수록 학습 누적. *"It learns your projects, auto-generates skills, and never forgets how it solved a problem."* |
| **Scheduled Automations** | *"Natural-language scheduling for reports, backups, and briefings — running unattended through the gateway."* |
| **Delegates & Parallelizes** | *"Isolated subagents with their own conversations, terminals, and Python RPC scripts."* |
| **Real Sandboxing** | 5종 백엔드 — *"local, Docker, SSH, Singularity, Modal — with container hardening and namespace isolation."* |
| **Full Web & Browser Control** | 웹·브라우저 조작 가능 (vision 포함) |

## 스킬 = 슬래시 커맨드 (핵심 개념)

[공식 퀵스타트](https://hermes-agent.nousresearch.com/docs/getting-started/quickstart)가 짚는 핵심 두 문장:

> "Each is a `SKILL.md` file with a name, a description, and a step-by-step procedure."
>
> "Every installed skill becomes a slash command automatically: `/k8s deploy`"

스킬 한 장(`SKILL.md` markdown)이 곧 슬래시 커맨드로 등록되는 구조. 라이브 본 행사([3. 사고 메커니즘을 스킬로 옮기기](/week4/skill))의 출발점이 바로 이 한 줄.

## OpenClaw와 비교 (공식 자기 정의 기준)

두 프로젝트의 *자기 정의 한 문장*을 그대로 두면 차이가 가장 명확합니다.

| 항목 | 헤르메스 에이전트 | OpenClaw |
|---|---|---|
| 자기 정의 (공식 원문) | *"An autonomous agent that lives on your server, remembers what it learns, and gets more capable the longer it runs."* | *"A personal AI assistant you run on your own devices. It answers you on the channels you already use."* |
| 한 줄 슬로건 | *"The Agent That Grows With You."* | *"Your own personal AI assistant. Any OS. Any Platform. The lobster way. 🦞"* |
| 메이커 | Nous Research | Peter Steinberger (`steipete`) |
| 라이선스 | MIT | MIT |
| 주 언어 | Python | TypeScript / Node.js (22.19+ 또는 24) |
| 첫 공개 | 2025-07-22 | OpenClaw 이전부터 운영 |
| 채널 통합 | Telegram · Discord · Slack · WhatsApp · Signal · Email · CLI | 20+ 채널 (WhatsApp · Telegram · Slack · Discord · Google Chat · Signal · iMessage · IRC · Microsoft Teams · Matrix · Feishu · LINE · Mattermost · Nextcloud Talk · Nostr · Synology Chat · Tlon · Twitch · Zalo · WeChat · QQ · WebChat 등) |
| 핵심 구조 강조 | *스킬* — `SKILL.md` 한 장이 곧 슬래시 커맨드 | *게이트웨이* — *"The Gateway is just the control plane — the product is the assistant."* |
| 데스크탑/모바일 부가 기능 | 5 백엔드 sandbox · Python RPC subagent | live Canvas · macOS/iOS/Android 음성 입출력 |

**두 자기 정의를 그대로 읽어보면** —
- 헤르메스: *server에 사는 자율 에이전트, 기억하고 자라남*
- OpenClaw: *본인 디바이스에서 도는 개인 어시스턴트, 본인이 쓰는 채널에서 응답*

**4회차에서는 헤르메스를 다루지만** OpenClaw도 같은 자리(로컬-퍼스트 · MIT · 다채널 통합)에 있는 선택지라는 점만 짚어둡니다. *어느 쪽이 본인에게 맞는지*는 라이브에서 본인이 직접 한 번 셋팅해보고 판단해보는 자리.
