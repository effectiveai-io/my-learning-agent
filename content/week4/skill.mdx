# 3. 내 학습 키워드를 SKILL.md로

> 메인 액티비티. 두 단계로 진행합니다.
>
> **(1)** 효과적이었던 내 학습 키워드 한 개를 에이전트와 대화하며 SKILL.md로 옮기기
>
> **(2)** 호출 · 평가 · 수정 · 재호출 사이클로 같은 키워드가 다른 학습 상황에도 반복 적용되게 다듬기

<Callout type="info">
**스킬 한 장이 곧 슬래시 커맨드** — 헤르메스의 핵심 컨셉. [공식 quickstart](https://hermes-agent.nousresearch.com/docs/getting-started/quickstart) 인용 두 줄:

**"Each is a `SKILL.md` file with a name, a description, and a step-by-step procedure."** — 각 스킬은 이름·설명·단계별 절차가 적힌 `SKILL.md` 파일 한 장.

**"Every installed skill becomes a slash command automatically: `/k8s deploy`"** — 설치된 모든 스킬은 자동으로 슬래시 커맨드가 된다.
</Callout>

## 1단계 — 내 학습 키워드 → SKILL.md

### 1) 본인 학습 키워드 고르기

라이브 전에 **스프레드시트**에서 각자 다음 3가지를 적어옵니다:

1. **지금 돌이켜 봤을 때, 나에게 효과적이었던 교육적 경험 3가지**
2. **지금 돌이켜 봤을 때, 나에게 그다지 효과적이지 않았던 교육적 경험 3가지**
3. **그 차이를 만든 키워드 3가지**

3번의 **키워드 3가지가 본인 학습의 핵심 변수**. 본인이 무언가 배울 때 또는 누군가에게 전할 때 효과를 좌우하는 그 단어들입니다. 라이브에서 이 중 **한 개**를 골라 에이전트 스킬로 옮깁니다 — 새 학습 상황에 들어갈 때마다 그 키워드가 자동으로 작동하도록.

<Callout type="note">
3번 키워드 예시 — 다른 사람과 하기 / 이벤트와 연결하기 / 현실적용 / 인풋보다 많은 아웃풋 / 피드백 등. 거창하지 않아도 됩니다. 본인 경험에서 반복적으로 차이를 만든 **본인 단어**가 곧 본인 키워드입니다.
</Callout>

### 2) 에이전트와 대화하며 SKILL.md 함께 만들기

본인 키워드 한 개를 들고 디스코드 본인 채널에서 에이전트에게 던집니다:

> **"내가 학습할 때 효과적이었던 키워드 한 개를 SKILL.md로 만들고 싶어. 키워드는 `{본인 키워드}`야. 이 키워드가 다른 학습 상황에도 반복 적용되도록 스킬로 함께 만들어줘."**

에이전트가 SKILL.md 초안을 돌려주면, 본인이 **한 줄씩** 검토하면서 본인 표현으로 바꿉니다 — 통째로 받지 않습니다. ([2주차 mechanism의 generation effect 원칙](/week2/mechanism))

**공식 예시** — 헤르메스가 따르는 [Agent Skills 공식 quickstart](https://agentskills.io/skill-creation/quickstart)의 `roll-dice` 스킬. 한 장 전부:

````markdown
---
name: roll-dice
description: Roll dice using a random number generator. Use when asked to roll a die (d6, d20, etc.), roll dice, or generate a random dice roll.
---

To roll a die, use the following bash command that generates a random number
from 1 to the given number of sides: `echo $((RANDOM % <sides> + 1))`

Replace `<sides>` with the number of sides on the die (e.g., 6 for a standard
die, 20 for a d20).
````

세 부분이 전부 — **name**(슬래시 커맨드 이름 `/roll-dice`), **description**(에이전트가 이 한 줄을 보고 호출 여부 판단 — 본인 키워드라면 본인이 **어떤 학습 상황에서 이 키워드를 적용하고 싶은지** 한 줄), **본문**(실제 절차).

본인 키워드 스킬도 같은 한 장:

```markdown
---
name: {커맨드명}
description: 본인이 {어떤 학습 상황}에 들어갔을 때 호출. {본인 키워드}를 적용함.
---

## 절차

1. 입력 — 본인에게 무엇을 받거나 어디서 읽어 오는가
2. 판단 — {본인 키워드}를 이 상황에 어떻게 적용하는가
3. 출력 — 본인에게 무엇이 한 줄로 도착하는가
```

### 3) 첫 호출로 동작 확인

저장하면 슬래시 커맨드로 자동 등록 — `/{커맨드명}`이 채널의 슬래시 자동완성에 뜨면 OK. 한 번 호출해서 결과를 본다.

**입력 가설**: 채널 슬래시 자동완성에 `/{본인 커맨드}`가 뜨고, 호출 시 결과가 돌아온다 → 2단계로. 안 뜨면 SKILL.md 형식·이름·저장 위치 점검.

## 2단계 — 학습 사이클

여기가 4회차의 핵심. 스킬은 **한 번 만들고 끝**이 아니라, **호출해서 결과를 보고 한 줄 고치는 사이클**로 자랍니다. 같은 키워드 스킬이 다른 학습 상황에도 잘 맞도록.

<Steps>
<Step title='호출'>
채널에서 `/{커맨드명}` 한 번 입력. 결과가 본인에게 어떻게 도착하는지 본다.
</Step>
<Step title='평가 — 한 줄로'>
**"이 결과가 내가 의도한 방향이 맞나? 본인 키워드가 이 상황에 맞게 작동했나?"** 맞으면 한 줄 메모, 아니면 어디가 어긋났는지 한 줄로 적기.
</Step>
<Step title='수정 — 한 번에 한 줄'>
SKILL.md의 절차 한 줄만 고친다. 두 줄 이상 고치면 무엇이 효과를 냈는지 모름 — **한 변수만** 바꾸는 게 핵심.
</Step>
<Step title='재호출'>
같은 입력으로 다시 호출. 결과가 더 본인 의도에 가까워졌는지 평가.
</Step>
</Steps>

사이클을 여러 번 반복합니다. 매번 한 줄 수정 → 재호출. 같은 키워드 스킬이 매 사이클마다 다른 상황에도 더 잘 맞도록 다듬어집니다.

<Callout type="tip">
🎯 본질 = **사이클을 본인이 굴려보는 것**. 스킬이 완벽해지는 게 아니라, **"이렇게 한 줄 고치면 결과가 이렇게 달라지는구나"** 라는 조정 감각을 본인 손으로 확인하는 작업.
</Callout>
