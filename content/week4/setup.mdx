# 2. 라이브 셋업 — 채널 입장과 첫 인사

> 라이브 첫 10분(19:30〜19:40). 본인이 해야 할 건 *디스코드 본인 채널 입장 + 한 줄 인사* — 그게 전부.

<Callout type="info">
**원활한 실습을 위해 사전 프로비저닝 방식으로 진행합니다.** CLI·데스크탑 설치는 환경별 변수가 많아 라이브 시간을 *셋업*에 소모하게 됩니다. 셋업 변수를 0으로 두고 라이브 시간 100%를 *학습 그 자체*에 쓰기 위해 — 운영자가 미리 깔아둡니다. 본인 환경 설치는 [미션](/week4/mission) 자리.
</Callout>

## 사전 프로비저닝 — 운영자가 미리 깔아둔 것

라이브 전에 운영자(메이커준)가:

- 참여자 1명당 헤르메스 에이전트 1개를 운영자 API key로 사전 설정
- 각 에이전트를 디스코드의 본인 전용 채널에 봇으로 할당
- 모델 provider · 64k 컨텍스트 · 기본 권한까지 모두 설정 완료

본인이 해야 할 환경 설정(Python·PATH·API key·Docker)은 **없습니다**.

## 1단계 — 디스코드 본인 채널 입장

라이브 시작 시간에:

1. 디스코드 챌린지 서버 접속
2. 본인 이름이 박힌 채널(예: `#agent-{본인}`)로 이동
3. 채널 안에 본인 에이전트가 봇으로 들어와 있는지 확인

**입력 가설**: 채널에서 본인 에이전트의 ready 신호(인사 또는 status 메시지)가 보인다 → 2단계로. 안 보이면 라이브 채팅으로 운영자에게 한 줄 알리기.

## 2단계 — 첫 한 줄 대화

채널에 한 줄 던집니다. 예시:

- *"안녕, 내 에이전트야. 너는 누구니?"*
- *"오늘 라이브에서 우리가 같이 할 일 한 줄로 정리해줘."*
- *"내 사용 가능한 슬래시 커맨드를 보여줘."*

**입력 가설**: 에이전트가 한국어로 한 줄 응답을 돌려준다 → 다음 단원([3. 사고 메커니즘을 SKILL.md로](/week4/skill))로. 응답이 부자연스럽거나 비어 있으면 라이브에서 운영자에게 알리기 (provider 또는 컨텍스트 점검).

<Callout type="tip">
🎯 10분의 본질 = *"내 에이전트가 거기 있다"* 한 줄을 본인 눈으로 확인하기. 셋팅 변수 없이 다음 35분 메인 액티비티로 즉시 들어가는 준비.
</Callout>
