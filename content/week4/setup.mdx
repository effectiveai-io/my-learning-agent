# 2. 로컬 셋팅 한 단계씩

> 한 번에 하나씩. 각 단계마다 **"이게 작동하면 무엇이 확인되나"** 라는 입력 가설을 같이 짚는다. 가설이 깨지면 거기서 진단한다 — 다음 단계로 넘어가기 전에.

<Callout type="tip">
🎯 **셋팅 끝지점**: 헤르메스 데스크탑 앱이 본인 머신에서 열리고, 본인이 정의한 스킬 1개가 슬래시 커맨드로 호출되는 시점까지.
</Callout>

## 왜 데스크탑인가 — CLI 대신 데스크탑으로 가는 이유

CLI(`pip install hermes-agent`)는 본인 머신의 Python 버전·PATH·셸 설정에 따라 환경별 변수가 너무 많습니다. 라이브 30분 안에 15명이 동시에 동작 가능한 상태로 만들기 어렵습니다.

**데스크탑 앱(`Hermes-Setup.dmg` · `Hermes-Setup.exe`)** 은:
- 클릭 한 번으로 설치 — Python·Node·PATH 셋팅 불필요
- macOS·Windows에서 동일한 UX
- provider 선택·스킬 추가·첫 chat 검증 모두 GUI

라이브에서는 **데스크탑 앱으로 진행**합니다. CLI는 4회차 미션 이후 본인이 환경에 맞춰 시도해보는 옵션.

## 0단계 — 사전 준비

- macOS **12 이상** 또는 Windows **10·11** ([공식 시스템 요구사항](https://hermes-agent.nousresearch.com/desktop))
- 컨텍스트 **64,000 토큰 이상** 사용 가능한 모델 ([Hermes 공식 요구사항](https://hermes-agent.nousresearch.com/docs/getting-started/quickstart))
- 본인이 평소 쓰는 API key 1개 (OpenAI·Anthropic·DeepSeek 등) **또는** 로컬 LM Studio·Ollama가 깔린 상태

**입력 가설**: 위 세 조건이 본인 머신에 다 준비됐다 → 1단계로. 빠진 게 있으면 거기부터 채운다.

<Callout type="warning">
**Linux 사용자**는 데스크탑 GUI가 제공되지 않습니다(공식 페이지: "Install via terminal"). 4회차 라이브는 데스크탑 기준이라, Linux 사용자는 CLI 경로를 별도 안내합니다.
</Callout>

## 1단계 — 데스크탑 앱 다운로드 + 설치

본인 OS에 맞춰 한 번 클릭:

| OS | 다운로드 |
|---|---|
| macOS | [Hermes-Setup.dmg](https://hermes-assets.nousresearch.com/Hermes-Setup.dmg) |
| Windows | [Hermes-Setup.exe](https://hermes-assets.nousresearch.com/Hermes-Setup.exe) |
| Linux | [터미널 설치 — 공식 안내](https://hermes-agent.nousresearch.com/docs/getting-started/quickstart) |

설치 흐름:
- **macOS**: `.dmg` 열고 Hermes 아이콘을 Applications로 드래그 → Spotlight(`⌘ + Space`)로 `Hermes` 검색해서 실행
- **Windows**: `.exe` 더블클릭 → 설치 마법사 따라가기 → 시작 메뉴에서 Hermes 실행

**입력 가설**: 앱이 열리고 시작 화면(welcome/onboarding)이 보인다 → 2단계로. 안 열리면 OS 버전·보안 차단(macOS Gatekeeper, Windows SmartScreen) 점검.

## 2단계 — provider 선택

데스크탑 UI에서 처음 만나는 단계 — 어떤 모델을 쓸지 정한다. 헤르메스는 40+ provider 지원이지만, 라이브에서는 본인이 평소 쓰는 한 곳만 연결.

선택지:
- **클라우드 API**: OpenAI · Anthropic · DeepSeek 등 (API key 입력)
- **로컬**: LM Studio · Ollama가 깔려 있다면 OpenAI-compatible endpoint URL 입력

<Callout type="info">
**한국어 응답 품질을 우선시한다면** — Claude Sonnet / GPT-4.x 계열이 안정적. 라이브에서는 본인이 평소 쓰는 한 곳으로 충분합니다.
</Callout>

**입력 가설**: provider 저장 후 메인 화면(채팅 UI)으로 진입한다 → 3단계로. 인증 에러면 API key 또는 endpoint URL 다시 확인.

## 3단계 — 첫 chat 검증

채팅창에 "안녕"이라고 쳐서 응답이 한 번 돌아오는지 본다. 라이브에서는 한국어 응답이 자연스러운 모델을 선택했는지도 같이 확인.

**입력 가설**: 응답이 한 번 돌아왔다 → 4단계로. 안 돌아오면 provider/모델 컨텍스트(64k 미만이면 헤르메스가 거부) 점검.

## 4단계 — 첫 스킬 추가

라이브에서 본인이 머릿속에 두고 쓰는 메커니즘 1개를 고른다. 예시:

- *"회고 한 줄 짚기"* — 7일 outbox를 읽고 반복된 패턴 1줄
- *"오늘 박을 송곳"* — 어제 outbox + `.ai-wiki`를 보고 오늘의 한 줄 의도

이걸 SKILL.md 한 장으로 옮긴다. 형식:

```markdown
# /회고

본인이 자주 부딪힌 패턴 1줄 짚기.

## 절차
1. 최근 7일 outbox 또는 노션을 읽는다.
2. 같은 단어·같은 좌절이 두 번 이상 나오면 그 한 줄을 뽑는다.
3. 그 한 줄을 "오늘 박을 송곳"으로 본인에게 1줄 보고한다.
```

데스크탑 앱에서 스킬을 추가하는 정확한 자리·경로는 라이브에서 화면 공유로 함께 짚습니다. (앱이 인식하는 skills 폴더에 위 markdown 파일 저장 → 앱 재기동 또는 새로고침)

**입력 가설**: `/회고`가 슬래시 커맨드로 인식된다(채팅창에서 `/` 입력 시 자동완성 후보에 뜬다) → 5단계로. 인식 안 되면 skills 위치·파일명·SKILL.md 형식 점검.
