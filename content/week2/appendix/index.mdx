## 한 문장 요약

<Callout type="info">
"질의 때마다 검색하는 RAG" 대신, **LLM이 마크다운 위키를 직접 만들고 매번 갱신**하는 패턴. 사람은 소스를 던지고 질문하고, LLM은 페이지를 쓰고 상호 참조를 유지한다.
</Callout>

## 3 레이어

| 레이어 | 누가 쓰나 | 내용 |
|---|---|---|
| **원본 소스** | 사람 | 기사·논문·노트·이미지. 절대 수정 X. 단일 진실 공급원. |
| **위키** | LLM | 마크다운 페이지. 요약·엔티티·개념·비교·종합. LLM이 전적으로 소유. |
| **스키마** | 사람+LLM | `CLAUDE.md` / `AGENTS.md`. 위키 구조·컨벤션·워크플로우 정의. 본인 도메인에 맞춰 같이 진화. |

## 3 동작

<Steps>
<Step title='Ingest · 소스 1개를 위키에 녹이기'>
LLM이 새 소스를 읽고, 본인과 핵심 시사점을 의논하고, 요약 페이지를 쓰고, 인덱스·엔티티·개념 페이지를 갱신하고, 로그에 항목을 덧붙입니다. 한 소스가 위키 페이지 **10〜15개**를 건드릴 수 있습니다.
</Step>
<Step title='Query · 위키에 질문하기'>
LLM이 관련 페이지를 검색해서 인용 포함 답변을 종합합니다. 출력은 마크다운, 비교 표, 슬라이드, 차트 등 질문에 따라 달라집니다. **좋은 답은 새 페이지로 위키에 다시 정리해 둡니다.** 탐색이 복리로 쌓이도록.
</Step>
<Step title='Lint · 위키 건강 검진'>
주기적으로 모순·고립 페이지(orphan)·낡은 주장·빠진 상호 참조·중요한데 자기 페이지 없는 개념을 점검합니다. LLM이 새로 조사할 질문과 새로 찾을 소스도 같이 제안합니다.
</Step>
</Steps>

## 두 개의 특별 파일

| 파일 | 정렬 | 역할 |
|---|---|---|
| `index.md` | 콘텐츠 중심 | 위키 전체 카탈로그. 인제스트마다 갱신. 질의 때 LLM이 먼저 읽음. **소스 100개·페이지 수백 개 규모까지는 임베딩 RAG 없이 인덱스 파일로 충분.** |
| `log.md` | 시간순 | append-only 기록. 일관된 prefix(`## [2026-04-02] ingest | Article Title`)를 박아 두면 `grep "^## \[" log.md | tail -5`로 최근 항목 추출. |

## 왜 작동하나

<Callout type="tip">
사람이 위키를 포기하는 이유는 **유지보수 부담**이다. LLM은 지치지 않고, 상호 참조 갱신을 잊지 않고, 한 번에 15개 파일을 건드린다. 유지보수 비용이 0에 가까워지니 위키가 살아남는다. 사람은 소스를 큐레이션하고, 분석 방향을 잡고, 좋은 질문을 던지고, 의미를 생각한다. 그 외 전부를 LLM이 한다.
</Callout>

이 아이디어는 정신적으로 [바네바 부시(Vannevar Bush)의 메멕스(1945)](https://en.wikipedia.org/wiki/Memex)와 연결됩니다. 문서 사이의 연상 경로를 가진, 개인적이고 능동적으로 큐레이션된 지식 저장소. 부시가 풀지 못한 부분은 *누가 유지보수를 하느냐*였습니다. 그걸 LLM이 처리합니다.

## 회차 메커니즘과의 매핑

2회차의 `in → 처리 → outbox` 메커니즘은 이 패턴의 **개인 학습 에이전트 인스턴스**입니다.

| LLM Wiki 레이어 | 우리 메커니즘 |
|---|---|
| 원본 소스 (raw) | `.inbox/` — 본인 PR, 노션 페이지, 노트 |
| 위키 | `outbox/` — 매일 아침 도착하는 메타인지 한 줄 + 누적되는 학습 페이지 |
| 스키마 | `.claude/commands/돌아보기.md` + `CLAUDE.md` — 어떤 흐름으로 LLM이 작업할지 정의 |
| Ingest | `/돌아보기` 커맨드 실행 |
| Query | "오늘 나는 뭘 모르고 있나?" 1줄 질문 |
| Lint | 다음 주 후보 — outbox/ 모순·고립 페이지 점검 |

원문은 *"본인 도메인에 맞춰 같이 인스턴스화하라"*고 명시적으로 말합니다. 우리 2회차는 그 인스턴스화 중 하나의 구체 사례.

## 출처와 라이선스

원문 출처는 명시되어 있지 않습니다. 본문에 *"본인의 LLM 에이전트에 복사해 붙여넣어 같이 작업하라"*고 명시되어 있어, 패턴 글로 자유롭게 공유·인용·재구성 가능한 의도로 공개된 문서로 봅니다. 라이선스가 별도로 표시되어 있지 않으므로, 인용 시 원문 링크 또는 본 부록 페이지로 출처를 표기합니다.
