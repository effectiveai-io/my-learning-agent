# 2. 나·LLM 저장소 메커니즘 한 장

> 1번 페이지의 **본인 케이스 매칭 1줄**을 들고 옵니다. 나와 LLM이 함께 읽을 저장소가 어떤 모양인지 — `in → 처리 → outbox` 흐름을 한 장으로 그립니다.

이 페이지는 라이브 3번 블록 · **15분**. 메인 액티비티 ①.

옵션 3 ("LLM과 나, 둘 다를 위한 지식 저장소") 5표를 다시 읽어보면 — 단순한 "둘 다"가 아니라 **나·LLM·저장소 3자가 협업하는 관계, 그리고 그 중심이 '나'** 라는 의미입니다. 오늘 라이브의 핵심 목표는 단 하나: **"라이브 끝나고 내 outbox에 글 1개 쌓는다."** 이 페이지는 그 한 글이 왜 중요한지, 그리고 어떤 형식으로 써야 하는지를 짚습니다.

## 1) 3자 협업 관계 — 나·Outbox·LLM, 나가 중심

내가 중심이 되어, 저장소와 LLM이 나를 돕는 협업 관계입니다. 저장소는 [김창준식 ACE 프레임워크](#ace) — Atlas(개념·주장) / Effort(현재 프로젝트) / Calendar(회고·액션) — 세 영역으로 분화시킵니다.

```mermaid
flowchart TB
    Me(["👤 나 (중심)<br/>약한 신호 감지<br/>의미 부여 · 의사결정"])

    subgraph Repo["📂 지식 저장소 = Outbox"]
        direction LR
        Atlas["🧭 Atlas<br/>개념 · 주장"]
        Effort["🛠 Effort<br/>현재 프로젝트"]
        Cal["📅 Calendar<br/>회고 · 액션"]
    end

    LLM(["🤖 LLM<br/>모순 찾기<br/>약한 신호 증폭"])

    Me -- "① 단락 글쓰기<br/>(스토리 · 첫 줄 주장)" --> Repo
    Repo -- "② 회상·인출<br/>(testing effect)" --> Me
    Me -- "③ 명령<br/>(모순? 약한 신호?)" --> LLM
    LLM -- "④ 후보 패턴 제시<br/>(라인 단위 채택)" --> Me
    Repo -- "⑤ substrate<br/>(평문·시간순·링크)" --> LLM

    classDef center fill:#fafce0,stroke:#b59f3b,stroke-width:3px,color:#5b4a0e
    classDef llm fill:#dee9fb,stroke:#3d6ff2,stroke-width:2px,color:#1f47b3
    class Me center
    class LLM llm
```

라이브에서 짚을 포인트 5개 (화살표 라벨 그대로):

1. **① 단락 글쓰기** — 팩트가 아닌 스토리, 첫 줄은 가장 강력한 주장 (definitional commitment).
2. **② 회상·인출** — 내가 다시 읽고 떠올리는 그 순간이 학습. 이게 testing effect.
3. **③ LLM에게 명령** — "모순 찾아줘", "약한 신호 뽑아줘" 두 가지 핵심 프롬프트.
4. **④ LLM 응답은 라인 단위로** — 단락 통째로 흡수 X. 한 줄씩 본인 노트로 옮길지 결정.
5. **⑤ LLM이 저장소를 직접 읽음** — 카파시식 substrate(평문·시간순·링크)라 가능. **이 한 화살표만 LLM이 저장소와 직거래**.

<Callout type="info">
**모든 의미부여는 '나'를 통과합니다.** 화살표 5개 중 4개가 "나"에서 출발하거나 "나"로 들어옵니다. LLM-저장소 직거래는 ⑤ 한 개. 즉 LLM은 보조 retriever이지, 작가가 아닙니다.
</Callout>

## 2) 왜 Outbox가 중심인가 — 학습은 input이 아닌 output에서 일어난다

[안드레 카파시](https://karpathy.github.io/)식 LLM wiki, [PARA](https://fortelabs.com/blog/para/), [Zettelkasten](https://en.wikipedia.org/wiki/Zettelkasten) — 방법론 이름은 다 다르지만, 학습과학에서 **단 하나로 수렴**합니다.

> **학습이 일어나는 곳은 input(읽기·저장)이 아니라 output(쓰기·인출)이다.**

```mermaid
flowchart LR
    R["📚 Retrieval Practice<br/>Roediger & Karpicke (2006)<br/>인출이 학습을 만든다"]
    G["✍️ Generation Effect<br/>Slamecka & Graf (1978)<br/>생성한 정보가 더 잘 남는다"]
    A["📝 Writing as Thinking<br/>Ahrens (2017) · Graham (2022)<br/>글쓰기 = 사고의 매체"]
    N["📖 Narrative Cognition<br/>Bruner · Klein<br/>서사로 저장되는 기억"]

    Out["📤 Outbox<br/>(매일 1단락 + 첫 줄 주장)<br/>= 사고가 일어나는 자리"]

    Think["🧠 내 사고가<br/>1점 더 깊어진다"]
    Act["🎯 다음 행동이<br/>바뀐다"]

    R --> Out
    G --> Out
    A --> Out
    N --> Out
    Out --> Think
    Out --> Act

    classDef src fill:#f7f6f3,stroke:#9b9a97,color:#3a3a37
    classDef out fill:#fafce0,stroke:#b59f3b,stroke-width:3px,color:#5b4a0e
    classDef result fill:#dee9fb,stroke:#3d6ff2,color:#1f47b3
    class R,G,A,N src
    class Out out
    class Think,Act result
```

라이브에서 짚을 포인트 3개:

1. **Testing Effect (인출 효과)** — [Roediger & Karpicke (2006)](https://psychnet.wustl.edu/memory/wp-content/uploads/2018/04/Karpicke-Roediger-2008_Sci.pdf)의 핵심 발견. 시험(인출)이 단순 재학습보다 장기 기억을 **1.5~2배** 강화. inbox에 100개 저장보다, outbox에 1줄 쓰는 게 학습 효과 큼.
2. **Generation Effect (생성 효과)** — [Slamecka & Graf (1978)](https://doi.org/10.1037/0278-7393.4.6.592). 직접 만든 정보가 받아 적은 정보보다 기억에 더 잘 남음. **LLM이 만들어준 요약을 그대로 붙이면 학습이 안 일어난다** (김창준 패치 노트 6.3과 정확히 일치).
3. **팩트가 아니라 스토리** — Bruner의 narrative cognition. "오늘 회의 있었다"가 아니라 "오늘 회의에서 OO가 한 말이 내 가정을 흔들었다 — 왜냐하면…". 결정의 순간·판단이 바뀐 순간·불편했던 대화를 1단락 서사로.

<Callout type="tip">
**Inbox 중심으로 생각하면 PIM에 빠진다** — 자료 수집·태그·폴더에 3개월. PKM은 **내가 쓴 1줄(outbox)이 본질**, inbox는 그 1줄을 위한 재료일 뿐. 오늘의 목표: "라이브 끝나고 내 outbox에 글 1개 쌓는다."
</Callout>

## 3) Outbox 한 글의 형식 — 나에게 좋은 것이 LLM에게도 좋다

> **같은 outbox 위에서 두 가지 retrieval이 일어난다** — 사람의 인출(testing effect) + LLM의 인출(하이브리드 서치). 한 형식으로 둘 다 만족시키는 게 옵션 3의 진짜 약속입니다. (출처: 김창준 패치 노트 6.4 *"두 가지 의미의 Retriever"*)

```mermaid
flowchart TB
    subgraph Form["📤 Outbox 한 글의 형식 (6원칙)"]
        direction LR
        F1["🧭<br/>첫 줄 =<br/>강력한 주장"]
        F2["📝<br/>한 단락<br/>한 아이디어"]
        F3["🔗<br/>위키 · 상대경로<br/>링크 (순환 OK)"]
        F4["📅<br/>YYYY-MM-DD<br/>prefix"]
        F5["📂<br/>평면 배치<br/>(no folder)"]
        F6["📖<br/>팩트 아닌<br/>스토리"]
    end

    Me["👤 나에게 좋음<br/>(김창준식 PKM)<br/>retrieval cue · 분화 · 창발"]
    LLM["🤖 LLM에게도 좋음<br/>(카파시 LLM wiki)<br/>context · 하이브리드 서치"]

    Form --> Me
    Form --> LLM

    classDef form fill:#fafce0,stroke:#b59f3b,stroke-width:2px,color:#5b4a0e
    classDef human fill:#fafce0,stroke:#b59f3b,stroke-width:3px,color:#5b4a0e
    classDef llm fill:#dee9fb,stroke:#3d6ff2,stroke-width:3px,color:#1f47b3
    class F1,F2,F3,F4,F5,F6 form
    class Me human
    class LLM llm
```

6원칙 대응표 — **같은 원칙이 양쪽 다 만족**:

| 형식 원칙 | 나에게 좋은 이유 (김창준식) | LLM에게도 좋은 이유 (카파시식) |
|---|---|---|
| 🧭 첫 줄 = 강력한 주장 | definitional commitment (Feynman) | LLM 컨텍스트 헤더 · 검색 결과 첫 줄 적중 |
| 📝 한 단락 한 아이디어 | atomic note (Luhmann) | RAG 청킹 친화 |
| 🔗 위키·상대경로 링크 | graph 구조, 순환 = 통찰 경로 | LLM이 따라갈 수 있는 평문 링크 |
| 📅 `YYYY-MM-DD` prefix | 시간순 = 서사 (Bruner) | LLM의 시간 인덱스 |
| 📂 평면 배치 | 해피 액시던트 (Unlinked Mentions) | 폴더 깊이 = 노이즈 |
| 📖 팩트 아닌 스토리 | narrative cognition | 맥락이 풍부한 청크 |

<Callout type="info">
**내 outbox가 곧 LLM 컨텍스트의 substrate** — 사람용 형식과 LLM용 형식이 충돌하지 않습니다. 같은 자리에서 만납니다. 이게 옵션 3 5표의 진짜 의미.
</Callout>
