# 3. 저장소 셋팅

> 앞 페이지에서 본 `outbox/` + `.ai-wiki/` 구조를, 본인 컴퓨터에 깝니다. **셋업 자체가 학습은 아닙니다.** 빨리 끝내고 [4번](/week2/outbox)에서 본인 데이터 만지러 가는 게 목적.

이 페이지를 따라가면 끝났을 때 본인에게 남아 있는 것은 세 가지입니다.

1. `~/git/my-brain-v1/` 폴더 (clone 완료)
2. `outbox/`에 본인 손으로 넣은 글 1개
3. `/돌아보기`를 한 번 돌려서 `.ai-wiki/`에 복제된 결과물 1개

이 셋이 안 보이면 셋업이 안 끝난 겁니다. 다음 페이지로 넘어가지 마세요.

## 1) 저장소 clone

starter 저장소 [`imakerjun/my-brain-v1`](https://github.com/imakerjun/my-brain-v1)을 clone합니다. 세 폴더(`outbox/` · `.inbox/` · `.ai-wiki/`)와 `.claude/commands/`까지 골격이 들어가 있어 손으로 만들 게 없습니다.

```bash
cd ~/git    # 또는 본인이 작업하는 폴더
git clone https://github.com/imakerjun/my-brain-v1.git
cd my-brain-v1
```

clone이 끝나면 폴더 구조가 이렇게 보입니다.

```
my-brain-v1/
├── outbox/                 # 사람용 영역 — 본인이 직접 쓰는 글 (append-only)
│   └── 첫-outbox.md        # 복사해서 본인 첫 글로 바꾸는 템플릿
├── .inbox/                 # raw 자료 격리 — 외부 자료 · AI 산출물 모두 던지는 곳
│   └── README.md
├── .ai-wiki/               # LLM용 자동 복제본 — /돌아보기·/수집이 채움
│   ├── README.md           # 폴더 의도 + LLM이 읽기 좋은 형식
│   ├── TEMPLATE.md         # frontmatter + 인용 단서 템플릿
│   └── index.md            # LLM이 Query 시 가장 먼저 읽는 카탈로그 (카파시 wiki 패턴)
├── .claude/
│   ├── commands/
│   │   ├── 돌아보기.md     # outbox → .ai-wiki 복제 + 메타인지 1줄
│   │   └── 주간정리.md     # 7일 누적 회고
│   └── settings.json       # 권한 · 기본 모델
└── README.md
```

<Callout type="tip">
**왜 점(`.`) prefix?** `.claude`처럼 점으로 시작하는 폴더는 Unix에서 **"내가 직접 안 건드리고 도구·커맨드가 관리하는 영역"** 이라는 의미가 박혀 있습니다. `.inbox/`(raw 자료 격리)와 `.ai-wiki/`(LLM 복제본)는 `/돌아보기`가 채워주는 자리, prefix 없는 `outbox/`만 본인이 매일 손으로 글을 쓰는 자리. `ls` 디폴트에서 `outbox/` 한 곳만 보이는 것도 의도된 우선순위 표현입니다.
</Callout>

여기서 **본인이 직접 쓰는 곳은 `outbox/` 한 곳**입니다. `.inbox/`는 raw 자료(외부 자료·AI 산출물 모두)를 던져두는 임시 폴더이고, `.ai-wiki/`는 손으로 안 만지고 커맨드가 채우는 LLM용 복제본입니다. 헷갈리면 [2번 메커니즘](/week2/mechanism#1-핵심-메커니즘--한-저장소에-두-영역)을 한 번 더 보세요.

## 2) 커맨드 둘러보기

`.claude/commands/` 안에 들어 있는 슬래시 커맨드를 한 번씩 열어봅니다. 라이브 후 본인이 직접 고치며 다듬어 갑니다.

| 커맨드 | 역할 |
|---|---|
| `/쓰기` | 새 글 한 편을 `outbox/`에 만듭니다. 두 가지(오늘 어떤 의도? · 파일명 키워드 한 단어?)만 묻고 6원칙 골격을 박아둠. 본문은 본인이 채움 |
| `/수집` | URL·텍스트·노션 페이지를 `.inbox/`에 raw 그대로 저장 + `.ai-wiki/`에 `type: raw`로 미러. 다음 글 쓸 때 LLM이 이 자료를 컨텍스트로 바로 잡음 |
| `/돌아보기` | `outbox/`의 최근 글을 읽고 → `.ai-wiki/`에 `type: reflected`로 frontmatter·메타인지·열린 질문을 더해 복제 + 다음 outbox 골격 한 편 생성 |
| `/주간정리` | 최근 7일 `outbox/` 글을 한 페이지로 누적 · 반복된 패턴 1줄 짚음. **결과물은 `.inbox/`에 둠** — AI가 만든 분석 자료라 사람 손맛 자리인 `outbox/`와 섞지 않음 |

네 커맨드가 한 사이클로 돌아갑니다 — `/쓰기`로 시작(빈 페이지 공포), `/수집`으로 재료 쌓고(빈 인박스 공포), 본인 손으로 본문 채우고, `/돌아보기`로 어제~오늘을 짚고(빈 동기화 공포), 일요일마다 `/주간정리`로 한 줄 패턴 뽑기. 라이브에서는 `/쓰기` · `/수집` · `/돌아보기` 세 개를 v1으로 같이 깔고, `/주간정리`는 미션 7일을 채운 다음 주에 본인이 처음 돌려봅니다.

<Callout type="tip">
**왜 `/쓰기`가 **질문 두 개**를 던지는가** — 빈 페이지 공포가 학습 시작의 최대 장벽입니다. 첫 줄 주장이 가장 어려운데, AI가 그걸 **대신 써주면** generation effect가 무너집니다. 대신 **"오늘 어떤 의도로 글을 쓰려고 해?"** 한 줄을 매일 묻게 하면 — 그 짧은 인출이 매일 같은 시간에 자동으로 던져지는 메타인지 트리거가 됩니다. [메타 룰](/week2/mechanism#outbox-경계--단위는-내-손-다듬기는-ai-손)대로 **AI를 출력자가 아니라 인출자로** 쓰는 가장 짧은 형태.
</Callout>

## 3) 셋업 검증 1줄

clone만 하고 다음으로 넘어가면 v1이 안 돕니다. **본인이 손으로 한 번 굴려보고 결과물이 눈에 보이는 시점까지** 셋업입니다.

<Steps>
<Step title='/쓰기로 첫 글 만들기'>
Claude Code에서 `/쓰기` 입력. 두 질문에 답하면 (예: 의도 **"PKM이 아니라 PIM에 빠진 것 같다"** · 키워드 **`pkm-함정`**) `outbox/{오늘}-pkm-함정.md`가 [6원칙](/week2/mechanism#3-outbox-한-글의-형식--6원칙) 골격으로 생깁니다. 셋업 검증용 입력은 이걸로 끝 — 본문은 라이브 후에 채워도 됩니다.
</Step>
<Step title='/돌아보기 한 번 돌리기'>
Claude Code에서 `/돌아보기` 입력. `.ai-wiki/`에 같은 이름의 파일이 frontmatter와 메타인지·열린 질문 두 줄이 더해진 채로 복제되는지 봅니다.
</Step>
<Step title='.ai-wiki에 파일 1개 떴는지 본인 눈으로 확인'>
`ls -a` 또는 에디터에서 `.ai-wiki/` 폴더 열기. 파일 1개가 보이면 셋업 끝. (`ls`만 하면 점 폴더는 안 보입니다 — 의도된 동작.) 안 보이면 권한 설정(`settings.json`) 또는 커맨드 본문에서 막힌 것. 라이브에서 같이 짚습니다.
</Step>
</Steps>

<Callout type="tip">
🎯 검증의 본질은 "동기화가 동작하는가"가 아니라, **본인이 `outbox/`에 첫 글을 한 줄 적었는가**입니다. 그 첫 줄이 오늘의 outbox 첫 데이터.
</Callout>

## 다음 주에 추가될 후보 (참고)

v1에는 안 들어갑니다. 본인이 outbox를 7일 굴려본 뒤, 막힌 지점이 보이면 그때 하나씩 깝니다.

| 이름 | 종류 | 역할 |
|---|---|---|
| `/약점짚기` | 커맨드 | outbox 최근 글을 AI가 **약점만 짚어줌**. "다시 써줘" X · "내가 약한 부분 짚어줘" O ([2번 메타 룰](/week2/mechanism#outbox-경계--단위는-내-손-다듬기는-ai-손)) |
| `korean-spell-check` | 스킬 | 한국어 맞춤법 다듬기. 단락 통째 재작성 X, 글자 단위 교정만 (호출은 "맞춤법 봐줘") |
| `humanizer` | 스킬 | AI 톤으로 빠진 문장을 본인 톤으로 되돌림 (메이커준 본인 글로벌 스킬. 호출은 "사람답게 · 내 톤으로") |

<Callout type="info">
스킬은 Claude Code 전역(`~/.claude/skills/`)에 깔리는 자산입니다. 학습 에이전트 한 저장소만의 것이 아니라 본인이 평소에 쓰는 도구 세트의 일부 — 한 번 깔면 다른 프로젝트에서도 같이 작동합니다.
</Callout>
