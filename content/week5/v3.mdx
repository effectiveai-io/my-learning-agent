# 4. v3 — 본인 환경으로 이관

> 라이브에서 만든 v1·v2가 운영자 인프라(헤르메스 에이전트 + 사전 셋팅 저장소)에서 동작했다면, v3에서는 그 구조를 **본인 LLM 환경**으로 옮겨 직접 운영해봅니다.

## v3의 본질 — 주도권 이전

운영자 인프라는 라이브 기간 동안만 동작합니다. 챌린지 이후에도 학습 리포트 루틴을 계속 굴리려면 **본인 통제 하에 v3가 굴러가는 상태**가 되어야 합니다.

v3는 v1·v2의 단순 복제가 아니라 본인 환경에 맞춘 **커스텀 버전**입니다. 본인이 평소 쓰는 LLM 도구, 본인의 자료 소스, 본인의 알림 채널에 맞춰 다시 짭니다.

## 본인 환경 선택지

평소 가장 자주 쓰는 환경에서 v3를 만듭니다.

| 환경 | 특징 | 적합한 분 |
|---|---|---|
| **Claude Code** | 로컬 CLI + 슬래시 커맨드 + skill | 터미널과 git 환경이 편한 분 |
| **Claude.ai Projects** | 웹 UI + custom instructions + 파일 업로드 | 웹 환경, 코드 안 짜고 싶은 분 |
| **헤르메스 데스크탑** | 4회차에서 깔아본 환경 | 데스크탑 앱이 편한 분 ([4회차 미션](/week4/mission) 이관 흐름 그대로) |
| **OpenAI Custom GPT** | ChatGPT Custom Instructions + Actions | 평소 ChatGPT를 쓰는 분 |
| **n8n / Zapier 자동화** | 노코드 자동화 + 스케줄링 | 자동화 도구가 익숙한 분 |
| **자체 스크립트** | Python · Node.js + cron + LLM API | 코드 직접 짜는 게 가장 편한 분 |

라이브 시간 안에 본인이 **오늘부터 정말 매일 쓸 수 있는 환경** 하나를 골라 v3를 시작합니다.

## 이관 4단계

### 1) v2에서 가장 효과적이었던 분석 골라내기

v2 실험에서 1〜2개 분석 옵션을 시도해봤을 겁니다. 그 중 **내 학습에 가장 인사이트를 준 것** 1개를 v3의 출발점으로 가져갑니다. v3에서는 **모든 옵션**이 아니라 **본인에게 가장 잘 맞는 한 가지**를 영구화해봅니다.

### 2) 본인 환경의 시스템 프롬프트로 옮기기

v1·v2가 디스코드 채널에서 자연어로 동작했다면, v3는 본인이 선택한 환경의 **영구 프롬프트**로 박힙니다.

- **Claude Code** → `.claude/commands/리포트.md` 같은 슬래시 커맨드
- **Claude.ai Projects** → 프로젝트의 system instructions
- **헤르메스** → SKILL.md 형태
- **Custom GPT** → instruction 필드
- **자체 스크립트** → 코드 안 시스템 프롬프트 상수

핵심은 **매일 자동으로 호출 가능한 자리**에 프롬프트를 두는 것입니다.

### 3) 본인 자료 소스 연결

v1·v2가 [learning-report 저장소](https://github.com/imakerjun/learning-report)를 읽었다면, v3는 본인 자료를 직접 읽어야 합니다.

- 본인 `outbox/`가 어디에 있는가 (GitHub repo · 노션 · 로컬 폴더)
- 본인 디스코드 채널 대화를 어떻게 export할까
- 본인 `SKILL.md` 모음은 어디에 있는가

각 도구마다 자료 접근 방식이 다릅니다. 본인 환경에서 **자료를 매일 자동으로 어떻게 읽어올지** 한 줄로 정합니다.

### 4) 스케줄 설정 + 도착 경로

매일 아침 자동 발송 설정. 본인 환경의 스케줄링 도구를 활용합니다.

- **Claude Code** → cron + 스크립트
- **Claude.ai Projects** → 수동 (Projects는 cron 미지원 → 별도 트리거 필요)
- **헤르메스** → 자연어 스케줄링
- **n8n · Zapier** → 시간 트리거 노드
- **자체 스크립트** → crontab · GitHub Actions

도착 경로는 본인이 **아침에 가장 먼저 여는 자리**로 정합니다 (메일 · 디스코드 DM · 노션 · 텔레그램 등).

## 검증

본인 머신에서 한 번 수동 트리거 → 리포트가 도착하는 형태가 v2와 비슷한가 → 매일 아침 자동 발송이 다음 날 정말 작동하는가.

**입력 가설**: 다음 날 아침 7시에 본인 환경의 v3 리포트가 도착한다 → 챌린지가 끝나도 학습 리포트 루틴이 본인 통제 하에 살아남은 상태입니다.

<Callout type="info">
**v3가 챌린지의 진짜 종착점** — v1은 운영자 인프라에서의 동작 확인, v2는 운영자 인프라에서의 실험, v3는 본인 환경으로의 이관. 데모데이에서 발표할 **4주 누적 한 단어**는 이 단어가 챌린지 끝난 후에도 자라는가가 핵심이고, 그건 v3가 본인 환경에서 굴러가야 가능합니다.
</Callout>
